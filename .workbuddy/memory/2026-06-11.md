# 2026-06-11

## Credits 充值申请系统 — 完整搭建

### 技术栈
- **前端**: 纯 HTML/CSS/JS（无框架，静态页面）
- **数据库**: Supabase (PostgreSQL)
- **Auth**: Supabase Auth（管理员邮箱密码登录）
- **部署**: 静态文件，可直接用本地文件或部署到任何静态托管

### Supabase 配置
- **项目 URL**: `https://iajpcsbebjjtaaspmwws.supabase.co`
- **表**: `public.applications`
  - id (uuid, PK), name, email, course, status (pending/approved/rejected), reject_reason, created_at, updated_at
- **RLS 策略**:
  - INSERT: 所有人可提交（学员表单）
  - SELECT: 所有人可查看（结果页查询状态）
  - UPDATE: 仅已认证用户（管理员后台操作）
- **管理员**: `admin@enter.pro` / `Admin123!`（通过 Supabase Auth）

### 页面结构
| 页面 | 路径 | 功能 |
|------|------|------|
| 学员申请表单 | `index.html` | 邮箱/名称/课程 → 提交到 Supabase → 跳转结果页 |
| 申请结果页 | `result.html?id=xxx` | 查看 pending/approved/rejected 状态，支持刷新和再次提交 |
| 管理员登录 | `admin/login.html` | Supabase Auth 邮箱密码登录 |
| 管理员工单列表 | `admin/dashboard.html` | 工单列表（按时间降序，分页20条），查看详情，通过/驳回操作 |

### 完整用户流程
1. 学员打开 `index.html` → 填写信息 → 提交
2. 跳转到 `result.html?id=xxx` → 显示"待处理"
3. 管理员登录 `admin/login.html` → 进入工单列表
4. 管理员点击工单 → 弹出详情 → 点击"通过"或"驳回"
5. 学员刷新结果页 → 看到状态已更新
